## 系统概述

**Aegis** 是一套运行在个人Windows电脑上的私人AI助理系统。它以用户真实的数字生活数据为基础——邮件、微信聊天记录、本地文件——通过AI分析将这些碎片化信息整合为结构化的个人知识记忆库，并支持通过邮件或微信远程向系统下达指令，实现随时随地的智能查询与任务控制。

Aegis不依赖任何云端存储，所有数据本地处理，所有记忆本地保存，兼顾智能与隐私。

### 系统定位

| 维度 | 描述 |
|------|------|
| 使用者 | 个人用户（单用户系统） |
| 运行位置 | 用户本地 Windows 电脑 |
| 核心价值 | 整合多源数据、构建个人记忆、支持远程指令 |
| 隐私策略 | 全本地处理，不上传原始数据 |
| AI 后端 | 火山引擎（豆包）大模型 API |

## 设计理念

### 1. 以真实数据为记忆基础

Aegis拒绝让用户从零手动录入信息。系统主动接入用户已有的数字资产——邮箱、微信、硬盘文件——从中自动提取有价值的知识片段，构建出真正属于用户本人的记忆库。

### 2. 导航地图优于内容副本

硬盘有数万个文件，微信有数十万条消息。**把所有内容压缩成摘要文件是错误的方向**——信息必然大量丢失，且维护成本极高。

正确做法是把文件系统理解为一个数据库：
- **文件夹名本身就是信息**：`博士论文写作/GPT_前列腺` 直接告诉了研究方向，无需读取内容
- **元数据索引**（路径+文件名+修改时间）比全量提取更实用
- **按需精读**：被问到某个话题时，再去找对应文件读内容，而不是提前把所有文件都压成摘要

### 3. 选择性深度处理

不是所有数据都值得向量化或AI分析。根据数据价值分级处理：

| 数据 | 处理方式 | 原因 |
|------|---------|------|
| 邮件 | 全量向量化 + AI联系人画像 | 无结构，语义搜索是唯一入口 |
| 微信消息 | FTS索引 + 三层记忆构建 | 高噪音，按联系人分组是关键 |
| 关键文档（CV/标书/论文） | 向量化 + AI提取 | 数量少（~100个），价值高 |
| 活跃代码（最近60天） | 向量化 | 了解项目进展 |
| 其他文件（全量） | 仅元数据索引 | 不读内容，靠文件名+路径导航 |

### 4. 渐进确认，保障记忆准确性

AI提取的信息不会直接写入记忆库，而是进入 **pending（待确认）队列**，由用户审核后才落库。确保记忆内容的准确性。

### 5. 月度校验，保持记忆新鲜

个人背景记忆每月自动合并邮件、微信中的新信息，保持与用户实际状态同步。每次重建时经过 AI 校验，去除他人信息，消除重复。

## 核心功能详解

### 1. 数据接入层

#### 1.1 邮件同步（双邮箱）

- 连接 **163邮箱** + **Gmail**，通过 IMAP/OAuth 批量同步历史邮件
- 规则快速分类（广告/重要/普通），只对重要邮件做 AI 精读
- 同步结果写入 SQLite `emails` 表
- 增量同步，每 30 分钟检查新邮件
- 同步后自动触发邮件记忆构建

#### 1.2 微信聊天记录解密导入

- 支持微信 **4.x 版本**（Weixin.exe），SQLCipher 4 加密
- 通过读取微信进程内存提取密钥（需管理员权限，一次性操作）
- 密钥保存后，后续同步**无需管理员权限**
- 正确识别本人发言（`real_sender_id` 为空 = 本人发送）
- 当前已导入：**492,588 条聊天消息**，**18,043 个联系人**

#### 1.3 硬盘文件扫描（元数据索引）

**不读文件内容**，只扫描：
- 文件路径、文件名、扩展名
- 最后修改时间 → 活跃度分层（热/温/冷）
- 从路径提取关键词，存入 SQLite 供搜索

活跃度分层：
| 层级 | 修改时间 | 用途 |
|------|---------|------|
| 🔴 热区 | 最近30天 | 当前工作焦点 |
| 🟡 温区 | 31-180天 | 近期项目 |
| ⚫ 冷区 | 180天以上 | 历史背景 |

---

### 2. 知识处理层

#### 2.1 选择性向量化

- **邮件**：全量向量化，支持语义搜索历史往来
- **微信**：FTS5 全文索引，支持关键词检索
- **关键文档**：CV、研究总结、国自然标书等约100个文件，向量化
- **活跃代码**：E:/codes 下最近60天修改的 .py/.md，向量化

停止全量向量化（原278k文件），改用元数据索引。

#### 2.2 FTS5 全文索引

- SQLite FTS5 + jieba 中文分词
- 覆盖：邮件 + 微信消息 + 关键文档
- 与向量索引互补：关键词精确匹配 + 语义模糊匹配

---

### 3. 指令通道

#### 3.1 邮件指令

向系统绑定邮箱发送邮件，标题包含 `Aegis:` 前缀，系统解析执行后邮件回复。

```
主题：Aegis: 查找上周关于国自然的邮件
主题：Aegis: 我最近在做什么项目？
主题：Aegis: 今日简报
```

#### 3.2 微信指令

向你在 `.credentials` 中配置的 `AEGIS_WXID` 账号发消息，以 `Aegis:` 开头，系统执行后邮件回复结果。

## 知识构建架构

### 硬盘知识构建（文件夹名优先策略）

```
第1步（秒级）：扫描目录树 → 只收集文件夹名 + 文件名 + 修改时间
                              ↓
第2步（分钟级）：AI 分析文件夹名 → 推断研究领域 / 重点目录 / 活跃状态
                              ↓
第3步：只精读 AI 识别的关键文件（CV / 研究基础整理 / 发表列表等，约20个）
                              ↓
第4步：AI 整合 → personal/background.md（已校验优化）
                              ↓
第5步（每月一次）：从邮件/微信新信息中合并更新个人背景
```

**关键洞察**：`博士论文写作/GPT_前列腺`、`国自然2026/听骨链微病变` 这些文件夹名直接揭示了完整的研究图景，比读文件内容快10倍、准确率更高。

### 邮件知识构建

```
邮件 DB（915封）
    ↓ 过滤广告/系统邮件（保留真实人类联系人）
    ↓
全局事实提取：从重要邮件推断用户相关信息（机构/项目/申请状态）
    +
联系人画像：每位真实联系人 → 身份/关系/往来事务/待跟进
    ↓
写入 from_emails.md + contacts/{addr}.md（重要性≥3）
```

### 微信知识构建（三层结构）

```
第一层：联系人档案（私聊，用户发言≥20次）
  → contacts/wx_{name}.md
  内容：身份/关系 · 主要话题 · 最近动态 · 待跟进

第二层：群聊档案（群聊，用户发言≥20次）
  → groups/{name}.md
  内容：群主题 · 核心成员 · 最近讨论

第三层：近期活跃事项（最近30天私聊）
  → wechat_active.md
  内容：用户的承诺/待办 · 对方在等待的事 · 截止时间

汇总：from_wechat.md（统计入口，链接各档案）
```

**过滤原则**：以**用户本人发言次数**为准（而非总消息数），确保只处理用户真正参与的对话，过滤单向广告/通知消息。

## 记忆系统设计

### 记忆目录结构

```
data/memory/
├── INDEX.md                  # 所有记忆的入口总索引
├── personal/                 # 个人背景记忆
│   ├── background.md         # 教育、职业、研究方向、论文、基金
│   └── background_backup_*.md  # 自动备份（每次更新前）
├── projects/                 # 项目记忆（E:/codes 每项目一个文件）
│   ├── Aegis.md
│   ├── 郑少婷.md
│   └── ...
├── contacts/                 # 联系人档案（邮件+微信）
│   ├── {email_addr}.md       # 重要邮件联系人
│   └── wx_{name}.md          # 重要微信联系人
├── groups/                   # 微信群聊档案
│   └── {group_name}.md
├── from_emails.md            # 邮件知识总览（联系人画像+事实）
├── from_wechat.md            # 微信知识总览（统计入口）
└── wechat_active.md          # 近期微信活跃事项（30天）
```

### 个人背景记忆（background.md）

- 来源：精读 CV、发表文章列表、国自然标书、个人介绍等**权威文件**
- 内容：基本信息、教育背景、工作经历、研究方向、代表论文、基金、当前动态
- 质量保证：AI 双步骤处理（先校验去除他人信息，再整合消除重复）
- 更新机制：每月自动从邮件/微信新信息中合并更新，附更新记录

### 项目记忆（projects/）

- 来源：E:/codes 下每个项目目录
- 内容：项目目的、当前状态、技术栈、关键功能、下一步
- 更新策略：7天内已有记忆则跳过，保持增量更新

### 联系人记忆（contacts/）

- 邮件联系人：重要性 ≥ 3 的真实联系人，单独建档
- 微信联系人：用户发言 ≥ 20 次的私聊对象，单独建档
- 群聊：用户发言 ≥ 20 次的群，单独建档

## 技术架构

### 架构总览

```
┌─────────────────────────────────────────────────────────┐
│                     Aegis                      │
├─────────────────────────────────────────────────────────┤
│  指令通道层        邮件指令 / 微信指令（AEGIS_WXID）         │
├──────────────┬──────────────────────────────────────────┤
│  数据接入层   │  邮件(163+Gmail) | 微信解密 | 文件元数据扫描 │
├──────────────┼──────────────────────────────────────────┤
│  知识处理层   │  选择性向量化 | FTS5全文检索 | AI记忆构建   │
├──────────────┼──────────────────────────────────────────┤
│  记忆系统层   │  个人背景 | 项目 | 联系人 | 群聊 | 活跃事项 │
├──────────────┼──────────────────────────────────────────┤
│  存储层       │  SQLite (jarvis.db) | ChromaDB | MD文件   │
├──────────────┴──────────────────────────────────────────┤
│  调度层           定时任务（APScheduler）/ 后台服务         │
└─────────────────────────────────────────────────────────┘
```

### 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 运行环境 | Windows 11 + Python 3.11 + Anaconda | 本地部署 |
| 主数据库 | SQLite（`jarvis.db`） | 邮件、微信、文件元数据索引 |
| 向量数据库 | ChromaDB（本地） | 邮件+关键文档语义搜索 |
| 全文检索 | SQLite FTS5 + jieba | 微信+邮件关键词搜索 |
| AI 模型 | 火山引擎（豆包）大模型 API | 分析、摘要、记忆提取 |
| 微信解密 | 自实现 SQLCipher 4 | AES-256-CBC + HMAC-SHA512 |
| 邮件协议 | IMAP (163) + OAuth (Gmail) | 双邮箱支持 |
| 任务调度 | APScheduler | 定时任务管理 |

### 数据库核心表

| 表名 | 内容 |
|------|------|
| `emails` | 邮件（发件人、主题、摘要、重要程度） |
| `wechat_messages` | 微信消息（内容、is_sender、会话ID） |
| `contacts` | 微信联系人（昵称、备注、wxid） |
| `file_index` | 文件元数据（路径、修改时间、活跃层级、关键词） |
| `memory_pending` | 待审核记忆条目 |
| `write_log` | 记忆写入日志 |

## 使用方式

### 命令行接口

```bash
# 系统初始化
python main.py --init

# 数据接入
python main.py --wechat                    # 导入微信（需管理员，首次）
python main.py --sync-emails --months 24  # 同步邮件（自动触发邮件记忆构建）

# 记忆构建
python main.py --build-memory              # 全量构建（个人+项目）
python main.py --personal-only             # 只重建个人背景
python main.py --projects-only             # 只重建项目记忆
python main.py --build-email-memory        # 重新提取邮件记忆
python main.py --build-wechat-memory       # 构建微信三层记忆
python main.py --build-wechat-memory --wx-top 50  # 只处理前50个最活跃联系人

# 硬盘索引（不读内容，仅元数据）
# 自动由每日 profile_update 任务执行，也可手动触发：
python -c "from scanner.file_metadata_indexer import scan_metadata; scan_metadata()"

# 向量化（仅关键文件）
python main.py --vectorize --batch 50

# 分析
python main.py --analyze-wechat --days 90  # 微信FTS索引 + pending提取
python main.py --focus-update              # 立即提取焦点事项
python main.py --briefing                  # 立即生成今日简报
python main.py --report                    # 生成状态报告

# 启动后台服务
python main.py --run
```

### 推荐初次部署步骤

```
1. python main.py --init               # 初始化数据库
2. python main.py --wechat             # 导入微信（管理员，微信登录中）
3. python main.py --sync-emails --months 24   # 同步邮件
4. python main.py --personal-only      # 构建个人背景记忆
5. python main.py --projects-only      # 构建项目记忆
6. python main.py --build-wechat-memory --wx-top 50  # 微信三层记忆
7. python main.py --run                # 启动后台服务
```
